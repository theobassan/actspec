# v0.1 — Composite actions

**Goal:** a test author can fully unit-test a `using: composite` action through the unified `mock` / `run` / `expect` surface, hermetically, with coverage. This is the foundation every later version extends.

## In scope
| Module | v0.1 slice |
|--------|----------|
| `@actharness/expressions` | already published at v0.0; v0.1 picks it up as a dependency — see [v0.0](v0.0.md) |
| `@actharness/types` | all public interfaces + `GITHUB_DEFAULTS` / `RUNNER_DEFAULTS`; zero deps — DAG root |
| `@actharness/core` | parser (`action.yml`), `ContextStore`, `RunnerProtocol`, `MockResolver`, executor registry, `ActionRunner`, result builder, error hierarchy, determinism |
| `@actharness/composite` | composite executor + `ShellSandbox` (real `bash` in a temp workspace) |
| `@actharness/matchers` | result + mock matchers; actharness's own `expect()` — no test-framework dependency |
| `@actharness/fixtures` | `github`/`runner` defaults + `push`/`pull_request`/`workflow_dispatch` event factories |
| `@actharness/coverage` | Istanbul-map collector + reporters (`text`,`html`,`lcov`,`json`,`cobertura`,`json-summary`); step · `if:`-branch · input/default · output layers; `coverageDir`, thresholds |
| `@actharness/cli` | `actharness test` (purpose-built runner: globals, coverage, parallel) + `actharness run` + `actharness init` |
| `actharness` | meta package re-exporting the common surface |

## Build order (dependency DAG)
0. **`@actharness/expressions`** — shipped at v0.0; install as a published dependency ([v0.0](v0.0.md)).
1. **`@actharness/types`** — zero dependencies. All public interfaces + `GITHUB_DEFAULTS`/`RUNNER_DEFAULTS`.
2. **`@actharness/core`** — depends on `types` + `expressions`. Parser → contexts → protocol → mock resolver → `ActionRunner` → results/errors.
3. **`@actharness/composite`** — depends on `core`. `ShellSandbox` + step loop (`if:` → run/uses → collect outputs) + action-output resolution.
4. **`@actharness/matchers`** — depends on `types` only (RunResult/StepResult/ActionMock/ShellMock).
5. **`@actharness/fixtures`** — depends on `types` only (GitHubContext/RunnerContext + defaults).
6. **`@actharness/coverage`** — depends on `types` only (consumes `RunResult`).
7. **`@actharness/cli`** — depends on `core` + `composite` + `matchers` + `coverage`.
8. **`actharness`** — re-exports.

## Integration checkpoint — the walking skeleton
After steps 1–3, **this test MUST pass** (it's the first end-to-end proof and the thing the whole design rides on):

```ts
// fixtures/greet/action.yml  (using: composite)
//   inputs: { name: { default: 'nobody' } }
//   outputs: { greeting: { value: '${{ steps.hello.outputs.greeting }}' } }
//   runs.steps:
//     - id: hello
//       run: echo "greeting=Hello ${{ inputs.name }}" >> "$GITHUB_OUTPUT"
//       shell: bash

// no imports — actharness, expect, describe, test are injected by `actharness test`

test('greet composite', async () => {
  const action = actharness('./fixtures/greet/action.yml');
  const result = await action.run({ inputs: { name: 'World' } });
  expect(result).toHaveSucceeded();
  expect(result).toHaveOutput('greeting', 'Hello World');
});
```

It exercises, in one shot: manifest parse, input defaulting + `INPUT_*`, expression eval (`inputs.*` and `steps.*.outputs.*`), `ShellSandbox` + real `$GITHUB_OUTPUT` protocol, composite output resolution, `RunResult`, and a matcher. If it passes, the architecture is validated; if not, we learn the gap before building wide.

## Additional v0.1 acceptance scenarios (in `@actharness/composite` + `@actharness/matchers`)
- **Mocked `uses:`** — `actharness.mock('actions/checkout@v4', { outputs: { ref: 'abc' } })`; assert `toHaveBeenCalledWith({...})` and that a later step saw the output.
- **Skipped step** — `if: ${{ inputs.publish }}` false → `toHaveStepSkipped('publish')`; and the branch shows uncovered until both paths are exercised (coverage).
- **Step failure** — a `run` step exits non-zero → `expect(result).toHaveFailed()`; with `continue-on-error: true` → run still succeeds, step `outcome: 'failure'`.
- **Step handle matchers** — `expect(result.step('id')).toHaveSucceeded()` / `.toHaveFailed()` / `.toHaveOutput(name, value)` / `.toHaveAnnotation({ level?, message? })` / `.toHaveStdoutContaining(s)` / `.toHaveStderrContaining(s)`; if `result.step('id')` returns `undefined`, all throw `"Expected step to exist, but step was not found"`.
- **`$GITHUB_ENV` threading** — step A writes env, step B reads it.
- **Local `uses: ./child`** — default `real` recursion runs the child composite; remote unmocked `uses:` → `noop` + warning (the [unmocked-uses policy](../../docs/ARCHITECTURE.md#mocking-model--two-distinct-surfaces-one-mental-model-mock-your-dependencies)).
- **Determinism** — two runs of the same action `toMatchSnapshot()` identically (frozen clock/seed).

## Implementation note

The following spikes produced working proof-of-concept code. **Do not rewrite from scratch** — copy and evolve that code into the corresponding packages:

| Spike | Source | Target package |
| --- | --- | --- |
| Runner (node:test) | `spike/runner/` | `@actharness/cli` + `@actharness/core` |
| API ergonomics | `spike/api-ergonomics/` (see `spike/workflow/src/` for evolved versions of `composite.ts`, `context.ts`, `types.ts`, `matchers.ts`) | `@actharness/composite`, `@actharness/matchers`, `@actharness/fixtures` |
| Coverage pipeline | `spike/coverage/` | `@actharness/coverage` |

**Important:** the workflow spike (`spike/workflow/`) directly evolved several api-ergonomics files. When implementing a package below, **check `spike/workflow/src/` first** — it is the more up-to-date version of that file:

| File | Why `spike/workflow/src/` supersedes `spike/api-ergonomics/` |
| --- | --- |
| `composite.ts` | Added `matrixCtx` parameter to `runComposite` |
| `context.ts` | Added `matrixCtx` to `buildContexts` / `buildEnvVars` |
| `types.ts` | `JobResult` uses `Omit<RunResult, 'conclusion'>` with wider `conclusion`; v0.1-blocking type fix |
| `matchers.ts` | `toHaveRunJob` checks `conclusion !== 'skipped' && !== 'cancelled'`; adds `toHaveSkippedJob`, `toHaveJobCancelled` |

`mock.ts`, `protocol.ts`, `parser.ts` were copied unchanged — either source is equivalent for those.

The spike code is the starting point; the module specs and acceptance criteria are the finishing gate.

## Exit criteria (v0.1 is "done")
1. Walking skeleton + all acceptance scenarios green.
2. v0.0 gate met — `@actharness/expressions` published, corpus green (443 vectors), fuzz in CI (see [v0.0](v0.0.md)).
3. Coverage reports (html/lcov/json/cobertura) emit; `if:`-branch truth table and output table work; 100% coverage threshold gates (`pnpm test:coverage`).
4. `actharness test` runs the test suite cleanly; matchers work via actharness's own `expect()` — no test-framework dependency.
5. `actharness run` executes a real composite from the CLI, matching the in-test path.
6. Every module meets its [CONVENTIONS Definition of Done](../../docs/CONVENTIONS.md#definition-of-done-every-module).
7. The public surface equals [API.md](../../docs/API.md) for the composite path (no shape owed to later versions).
