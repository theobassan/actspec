# v0.3 — Docker actions

> **Light stub.** Detailed module spec lands **once v0.2 is validated** (depth policy — [specs/README](../README.md#depth-policy)).

**Goal:** test `using: docker` actions through the **same** surface, **without requiring a Docker daemon** by default.

## Adds

- **`@actharness/docker`** — the docker `ActionExecutor` + `ContainerSandbox`.
- **No public API change** — `container` backend already in [API.md §1](../../docs/API.md) `ActharnessOptions`.

## The genuinely hard parts

- **No-daemon default** — a docker action reached via `uses:` is just another **mocked dependency** (declare outputs); CI without Docker still runs the suite. Real execution is opt-in (`container: 'docker' | 'podman'`).
- **Three image sources** — `image: Dockerfile`/`./path` (built on demand, content-hash cached) and `docker://registry/img` (pulled); normalized by `ContainerSandbox`.
- **Container wiring** — mount the protocol files and pass `args:`/`env:`/`entrypoint:` so a real container reads `$GITHUB_OUTPUT` exactly as on a runner.
- **pre-entrypoint / post-entrypoint** lifecycle (the docker analog of pre/main/post).

## Coverage gained

None new — docker actions are covered as **steps** (the existing step layer).

## Acceptance (high level)

A docker action **mocked** (default, no daemon) returns its declared outputs; with `container: 'docker'`, a real container builds/runs and its `$GITHUB_OUTPUT` is read back; the three image sources resolve.

All new packages meet the [CONVENTIONS Definition of Done](../../docs/CONVENTIONS.md#definition-of-done-every-module) — including 100% coverage (`pnpm test:coverage`): statements, branches, functions, and lines.
