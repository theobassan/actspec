# v0.0 ‚Äî `@actharness/expressions` standalone

**Goal:** publish the GitHub Actions expression engine as an independently useful npm package before the testing framework ships. This is the only deliverable at v0.0 ‚Äî no test runner, no action execution, no mocking surface.

## Why a standalone milestone

The expression engine is the riskiest and most independently valuable component:
- **Riskiest** ‚Äî "we match the C# runner" is a claim that can only be falsified by running the full conformance corpus green. That gate should be proven before the rest of the framework depends on it.
- **Independently valuable** ‚Äî no complete open-source JS implementation of the GitHub Actions expression language exists. The community needs this regardless of whether they adopt actharness's test runner.
- **Pipeline proof** ‚Äî publishing `@actharness/expressions` establishes the monorepo ‚Üí npm publish pipeline, CI fuzz gate, and API-extractor snapshot before any further packages land.

## In scope

| Deliverable | Detail |
|---|---|
| **`@actharness/expressions`** | Full public API: `evaluate`, `evaluateTemplate`, `ExpressionError`, `ExpressionContexts` |
| Dual ESM/CJS build | Consumers can use named imports in both module systems |
| API Extractor snapshot | Public surface locked; any breaking change fails CI |
| Full corpus green | All 443 vectors in `corpus/expressions/` pass |
| Parser/eval fuzz in CI | Differential fuzz against the documented grammar; non-blocking vs `nektos/act` oracle |
| `hashFiles` real implementation | Glob + SHA-256 per the runner spec; overridable via the `functions` hook |

## Out of scope (all belong to v0.1+)

- `@actharness/core`, `@actharness/composite`, or any other package
- The test runner, CLI, matchers, coverage
- The conformance corpus for protocol / runner behavior (those pin runner fidelity, not expression fidelity)

## Gate (v0.0 is "done")

1. `@actharness/expressions` published to npm.
2. Full corpus green: all 443 vectors in `corpus/expressions/` pass ‚Äî including the complete `&&`/`||` type matrix and all function test cases from `nektos/act`.
3. Parser/eval fuzz in CI (green, non-blocking vs act oracle).
4. API Extractor snapshot committed; breaking-surface CI check passing.
5. Dual ESM/CJS build verified (`@arethetypeswrong` clean).
6. 100% coverage enforced in CI (`pnpm test:coverage`): statements, branches, functions, and lines must all reach 100% ‚Äî no suppression annotations permitted.
7. `@actharness/expressions` meets the [CONVENTIONS Definition of Done](../../docs/CONVENTIONS.md#definition-of-done-every-module).
