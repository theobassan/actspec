# v0.4 — Workflows

> **Light stub.** Detailed module spec lands **once v0.3 is validated** (depth policy — [specs/README](../README.md#depth-policy)). This is the largest version, but still additive.

**Goal:** test whole **workflows** and **reusable workflows** (`on: workflow_call`) — through a parallel entry that reuses everything below the `StepRunner`. The `actharness()` action surface is **untouched**.

## Adds

- **`@actharness/workflow`** — the `WorkflowRunner` **orchestrator** (a sibling above the executor registry, *not* an executor — a workflow has no `runs.using`).
- **New public surface (the only version that adds any):** `actharnessWorkflow()`, `mockJob`/`mockReusable`/`mockService`, `wouldTrigger`, and `toHaveRunJob`/`toHaveJobConclusion`/`toHaveJobOutput` ([API.md §10](../../docs/API.md)). Existing step/output/mock matchers apply per job.
- `@actharness/coverage` gains the **`jobs`** layer (jobs run + `needs:` edges taken).

## Why it stays additive

A job's `steps:` are identical in shape to a composite's, so the expression engine, runner protocol, sandboxes, mock resolver, and `StepRunner` are **reused unchanged** — which only works because they were kept action-agnostic in v0.1 ([Future-proofing invariants](../../docs/ARCHITECTURE.md#future-proofing-invariants)). See [Workflow orchestration](../../docs/ARCHITECTURE.md#workflow-orchestration-v0.4).

## The genuinely new parts

- **Job DAG** — resolve `needs:` → topo order, run sequentially, thread `needs.<job>.outputs`/`.result`.
- **Matrix** — expand `strategy.matrix` (+`include`/`exclude`); `fail-fast`/`max-parallel` modeled as an *effect*.
- **Job environment** — `container:` (via `ContainerSandbox`) and `services:` (mocked via `mockService`).
- **Reusable workflows** — as unit-under-test (typed `inputs`, `secrets: inherit`, `outputs` wiring) and as a mocked dependency (`mockReusable`); nesting depth + cycle limits.
- **Trigger evaluation** — `wouldTrigger(event)` over `on:` filters (`branches`/`paths`/`tags`, `types`, `schedule`, `workflow_run`).
- **Metadata contexts** — `permissions`/`concurrency`/`environment`/`defaults` represented; `environment` approvals stay out of scope.

## Acceptance (high level)

Jobs run in `needs` order; `mockJob`/`mockReusable`/`mockService` replace dependencies; `wouldTrigger` reports which jobs a given event fires; a reusable workflow is tested standalone with inherited secrets; per-job assertions reuse the v0.1 matchers verbatim.

All new packages meet the [CONVENTIONS Definition of Done](../../docs/CONVENTIONS.md#definition-of-done-every-module) — including 100% coverage (`pnpm test:coverage`): statements, branches, functions, and lines.

**Two v0.1-blocking findings from the spike (apply before v0.1 ships):**

- `JobResult` must use `Omit<RunResult, 'conclusion'>` with `conclusion: 'success' | 'failure' | 'skipped' | 'cancelled'`
- `toHaveRunJob` must check `conclusion !== 'skipped' && conclusion !== 'cancelled'` (not just `!== 'skipped'`); `'cancelled'` arises from fail-fast — see spike code `spike/workflow/src/matchers.ts:110`
